# 项目路线图 (Roadmap)

本文档概述了草梅 Auth 的中长期发展方向和大型功能规划。

## 阶段 1: 稳定性、规范化与核心增强 (当前阶段)

_目标：通过重构和技术规范升级，打造稳健、安全的统一认证基石。_

-   [x] **架构解耦 (G1)**: 邮件、短信、验证码模块完成解耦与 DI 引入。
-   [x] **文件瘦身 (G2)**: 超长文件清理，逻辑下沉至 Composables。
-   [x] **Zod 基础设施**: 引入 Zod 实现 Schema 驱动校验。
-   [ ] **多语言支持 (i18n)**: 实现 Web、Email、API 的全量翻译支持。
-   [ ] **测试覆盖率达标**: 核心逻辑 Lines 覆盖率达到 60% 以上。

## 2. 阶段 2: 开发者生态与集成优化

_目标：提升集成便利性，让开发者更快接入。_

-   [ ] **轻量化 SDK**: 为 Web、Node.js 提供原生集成 SDK。
-   [ ] **文档中心自动化**: 完善 API 文档，支持在线调试接口。
-   [ ] **插件/钩子系统**: 支持在认证关键节点注入自定义逻辑。
-   [ ] **更多社交登录**: 集成飞书、企业微信等办公场景登录方案。

## 阶段 3: 安全增强与智能化管理

-   [ ] **SDK 开发**: 提供 JS/TS, Python, Go 等主流语言的轻量级集成 SDK。
-   [ ] **插件系统**: 支持开发者编写自定义逻辑插件（如：自定义登录校验）。
-   [ ] **更多社交登录**: 集成钉钉、飞书、企业微信等国内办公场景登录。

## 阶段 3: 安全增强与智能化管理

_目标：通过高级安全审计与智能分析提升平台防御能力。_

-   [ ] **风险管控体系**: 基于地理位置、设备指纹的风险分析。
-   [ ] **安全告警系统**: 异地登录、高频失败、异常 IP 实时告警。
-   [ ] **高级审计日志**: 满足企业级合规性要求的细粒度审计日志导出。

## 阶段 4: 企业级特性与 SaaS

_目标：面向大型组织和云原生环境提供多租户支持。_

-   [ ] **多租户架构 (SaaS)**: 单实例支持管理多个独立的组织租户。
-   [ ] **组织/部门管理**: 支持多级组织架构及权限委派。
-   [ ] **LDAP/AD 集成**: 与传统企业目录服务对接。
